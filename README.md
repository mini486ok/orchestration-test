# 🚆 Rail-Brain Test Lab

**철도·교통 분야 MCP(Model Context Protocol) 오케스트레이션 기술 개발·테스트 플랫폼**

테스트용 MCP 서버(메타정보 + 스키마 기반 모의 실행)를 등록하고, 프롬프트 엔지니어링·스킬·룰·DB 기반의
다양한 오케스트레이션 전략을 설계한 뒤, 분야별 벤치마크로 성능을 다각도 지표로 정량 검증·비교하는 웹 애플리케이션입니다.

**배포 주소**: https://mini486ok.github.io/orchestration-test/

## 주요 기능

| 기능 | 설명 |
|---|---|
| 🧩 MCP 카탈로그 | 철도·교통 샘플 MCP **100종**(10개 분야 × 10개, 도구 300여 개) 기본 제공. 검색·필터·상세 조회·삭제·복원, 스키마 기반 도구 직접 실행 |
| 🛠 MCP 빌더 | 수동 폼(스키마 직접 정의) + AI 자동 생성(텍스트 설명 → LLM이 서버 설계) |
| 🧠 오케스트레이션 스튜디오 | **4가지 전략** — 프롬프트(플랜/ReAct)·스킬·룰·**DB**(Vector/Graph) 워크플로우 설계·테스트·저장/불러오기 |
| ⚡ 실시간 테스트 | 여러 전략을 동시에 실시간 질의·비교. 자유 질의 + **벤치마크 문항 선택 시 정답 대조·채점** |
| 📏 벤치마크 랩 | 분야별 검증 세트 **11종·110문항**(기본 1 + 분야별 10). **다중 정답** 지원(순서 무관·대안 워크플로우·목표 도구). LLM 자동 생성 + 수동 작성, **세트 조합**, 가져오기/내보내기 |
| 🏁 평가·비교 | 다중 전략 동시 평가, **오케스트레이션 종합점수** + 다각도 지표, 차트·레이더 시각화, 지표 **용어집**, 실행 이력, CSV/JSON 내보내기 |
| 🔐 접근 제어 | 최초 실행 시 관리자 계정 생성(브라우저 로컬 저장), 개별 계정 로그인 |

## 오케스트레이션 전략 4종

- **프롬프트** — 전체 카탈로그 정보를 프롬프트에 넣고 계획(Plan) 또는 ReAct 방식으로 도구 호출.
- **스킬** — 도메인 스킬셋(트리거 + 도구 묶음)으로 라우팅.
- **룰** — 키워드/정규식 조건 기반 결정적 워크플로우.
- **DB** — MCP 카탈로그를 **검색용 DB로 색인**해 질의에 관련된 소수 도구만 공급(대규모 카탈로그에서 토큰 효율적).
  - **Vector DB** — 도구 임베딩(bge-m3) 인덱스. 임베딩 문서 구성(설명/파라미터/출력스키마/예시/태그 토글)·MMR 다양성(λ)·검색 방식(vector/keyword/hybrid) 등 파라미터 조절.
  - **Graph DB** — 도구 관계 그래프(io 입출력스키마·의미 임베딩·서버·카테고리·공출현·LLM추출 6종 엣지). 엣지 on/가중치/임계값·hop·노드 최대 연결(maxDegree)·허브 정규화·워크플로 경로 추천(빔 폭/길이/엣지)을 조절. **SVG 시각화**(줌·팬·확대, 노드/엣지 **호버 설명 툴팁**), 검색 미리보기, 경로 추천 UI 제공.

## 평가·비교 지표

전 지표는 화면 내 **용어집 패널**과 컬럼 툴팁에서 의미·계산식·해석을 확인할 수 있습니다.

- **오케스트레이션 종합점수** = `0.85 × 품질 + 0.15 × 토큰효율` — 리더보드 기본 정렬 기준(동일 실행 내 상대 지표).
  - 품질 = `0.4·F1 + 0.3·목표달성 + 0.15·도구성공률 + 0.15·파라미터정확도` (N/A 항목은 가중치 제외·재정규화).
- **F1 / 정밀도 / 재현율** — 기대 vs 실제 도구 집합 일치도.
- **목표 달성률** — 기대 워크플로우의 최종(목표) 도구를 오류 없이 호출(파라미터 지정 시 절반 이상 일치). 순서 무관 문항은 정답 도구 전부 완수.
- **도구 호출 성공률** — 호출한 도구 중 입력 스키마 검증·실행에 성공한 비율(낮으면 파라미터를 잘못 만들어 호출 실패).
- **잉여 도구 호출률** — 정답에 없는 불필요한 도구를 부른 비율(= 1 − 정밀도).
- **시퀀스 정확도 / 완전일치율 / 파라미터 정확도** — 순서·정확 일치·파라미터 값 채점.
- **입력/출력 토큰량** — Ollama 실측(`prompt_eval_count`/`eval_count`). 프롬프트 전략의 고토큰 vs DB 전략의 저토큰을 정량 비교.
- **평균 지연 / LLM 호출 수 / 오류율 / 폴백율** 등 실행 비용·안정성.

> 도구 실행은 모의(스키마에 맞는 결정적 랜덤 출력)이므로, "정답"은 자연어 답변이 아니라 **올바른 도구 워크플로우(+파라미터)**로 정의합니다. 한 질의에 여러 정답(순서 무관, 대안 경로)이 인정됩니다.

## 아키텍처

- **순수 정적 SPA** — 프레임워크/빌드 도구 없음 (Vanilla JS ES Modules). GitHub Pages 직배포.
- **LLM** — 브라우저에서 로컬 [Ollama](https://ollama.com) API 직접 호출 (기본 채팅: `exaone3.5:7.8b`, 임베딩: `bge-m3`, 변경 가능).
- **MCP 모의 실행** — 실제 서버 없이 inputSchema 검증 → outputSchema에 부합하는 결정적 랜덤 출력 생성.
- **DB 색인** — 브라우저 내장 경량 인덱스: Vector(임베딩·BM25·하이브리드 검색) / Graph(도구 관계 그래프 순회·경로 추천).
- **데이터** — 전부 브라우저 localStorage (JSON 백업/복원 지원). 서버 전송 없음.
- **(선택) 게이트웨이 서버 모드** — 다중 PC 공유·중앙 계정·계정별 LLM 쿼터가 필요하면 `server/`의 Python 게이트웨이 사용(`?gateway=<url>`). 자세한 내용은 [`server/README.md`](server/README.md).

## 시작하기

1. **Ollama 준비** (이 페이지를 여는 PC에서):
   ```powershell
   # 채팅 모델 + 임베딩 모델(DB 전략용) 설치
   ollama pull exaone3.5:7.8b
   ollama pull bge-m3
   # GitHub Pages(https)에서 접근 허용 (PowerShell, 1회)
   [System.Environment]::SetEnvironmentVariable("OLLAMA_ORIGINS", "*", "User")
   # 이후 Ollama 재시작
   ```
2. 배포 페이지 접속 → 최초 실행 시 관리자 계정 생성 → 로그인.
3. 앱 내 **가이드** 메뉴에서 상세 사용법을 확인하세요.

> 일반 브라우저 탭에서 Chrome의 "로컬 네트워크 액세스" 허용이 필요할 수 있습니다.

### 로컬 실행

```bash
git clone https://github.com/mini486ok/orchestration-test.git
cd orchestration-test
python -m http.server 8000   # 또는 아무 정적 서버
# http://localhost:8000 접속
```

## 디렉토리 구조

```
index.html
assets/
  css/main.css          # 디자인 시스템 (관제센터 다크 테마)
  js/
    app.js              # 엔트리 (인증 게이트 → 셸 → 라우터)
    core/               # store, router, ui, auth, charts
    services/           # ollama, mockEngine, orchestrator, benchmarkGen,
                        #   evaluator, catalogIndex(Vector), catalogGraph(Graph)
    data/               # 샘플 MCP 100종(sampleMcps + mcpsExt/*),
                        #   샘플 전략, 벤치마크 11세트(samples + benchmarksExt/*)
    views/              # 화면별 뷰 모듈(오케스트레이션·평가·실시간 테스트·벤치마크·가이드 등)
docs/                   # 아키텍처 명세(SPEC / SPEC-GRAPH / SPEC-GATEWAY)
server/                 # (선택) Python 게이트웨이 — 다중 PC·중앙 계정·쿼터
```

## 보안 관련 주의

정적 호스팅 특성상 로그인은 **클라이언트 측 접근 제어**입니다(PBKDF2 해시, localStorage 저장).
서버 수준의 보안이 아니므로 민감한 데이터는 저장하지 마세요.
계정 정보는 저장소(repo)에 포함되지 않으며 각 브라우저(또는 게이트웨이 서버 모드의 `server/data/`)에만 존재합니다.
